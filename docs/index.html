"What does Hydrogen do for me?"

Hydrogen is a design and development tool that helps you quickly build web projects that are mobile-ready, consistent, and accessible.

Component Library

Hydrogen offers over 25 customizable components that you can use to build your interfaces. Each component is built in HTML, Sass, and JavaScript, and is self contained so that they can be individually used if you don't have the need for the entire system.

Utility Attributes

Known in Hydrogen as "properties", the system includes over a dozen media query enabled data-attribute utilities that can be used to style individual elements. Leveraging custom data-h2 attributes, you can style anything from background colors to font sizes.

Theming via Sass

When you import Hydrogen into your Sass-enabled project, you can customize over 90 theme variables to ensure that your components and properties match your brand's guidelines. There are even a handful of pre-built themes on the way that you can use out of the box.

"Why is Hydrogen different?"

Hydrogen attempts to remain bias-free and technology agnostic while offering powerful control for any project.

In the interest of allowing designers and developers to use the technologies that best suit their project and users, Hydrogen is built using the basic building blocks of the web: HTML, CSS (with help from Sass!), and JavaScript. The system can be installed on any project and is namespaced in a way that allows you to install it worry free - it will not affect any of your existing work until you tell it to.

Hydrogen's nomenclature is designed to be explicit yet concise. It works to balance size, speed, and customization with clarity and plain language.

Each piece of Hydrogen's system can be used as a standalone feature. If your project needs accordion components, for example, you can simply install Hydrogen's accordion to avoid bloating your project with unnecessary code. This modularity allows Hydrogen alleviate the ever-present issues that arise with updating to a new major version - if you don't have the resources to bugfix breaking changes, you can install a previous version of a component alongside the most recent update to ensure continuity until you're ready to upgrade.

Get Started with Hydrogen

Please keep in mind that the technology behind Hydrogen is still in alpha. You might encounter bugs.

Hydrogen can be used as standalone CSS and JS files imported into your project, or if you'd prefer to customize the theme, you can import it into any Sass enabled project using NPM.

To install Hydrogen without customizing it's look and feel, simply include the following in your head element:

Hydrogen's CSS:

https://cdn.jsdelivr.net/gh/hydrogen-design-system/system/dist/cdn/h2-system.min.css

Cash and Hydrogen's JavaScript:

https://cdn.jsdelivr.net/gh/hydrogen-design-system/system/dist/cdn/cash/cash.min.js 
https://cdn.jsdelivr.net/gh/hydrogen-design-system/system/dist/cdn/h2-system.min.js

Once successfully imported, you can use any of Hydrogen's components. If you'd like to use its property utilities, ensure that you include data-h2-props on your body tag, or any element that wraps the elements you'd like to customize.

To get started with Hydrogen on your Sass project, begin by installing it via NPM:

npm install @hydrogen-design-system/system@0.0.19 --save-dev

Once intalled, you can load Hydrogen at the top of your project's Sass file using the following code:

@use "../node_modules/@hydrogen-design-system/system/dist/import/core/defaults" with (
    // Put custom variables here.
);
@use "../node_modules/@hydrogen-design-system/system/dist/import/h2-system.scss";

Learn more about what custom variables are available.

Next, you'll need to include Hydrogen's JavaScript:

https://cdn.jsdelivr.net/gh/hydrogen-design-system/system/dist/cdn/cash/cash.min.js 
https://cdn.jsdelivr.net/gh/hydrogen-design-system/system/dist/cdn/h2-system.min.js

Lastly, include data-h2-core and data-h2-props on your page's body element, or the elements you'd like to use Hydrogen with specifically.

You'll want to ensure that your project compiles Sass with Autoprefixer to ensure the best compatibility possible.
